This guide walks you through implementing a basic Checkout payment flow using the PayMaya Android SDK v2. You'll learn how to initialize the SDK, create a checkout request, and handle payment results.

## Prerequisites

<Check>
  Ensure you've [installed the SDK](/installation) in your Android project.
</Check>

You'll also need:
- A PayMaya client public key (use Sandbox key for testing)
- An Android Activity to host the payment flow

## Initialize the Checkout client

First, create a `PayMayaCheckout` client instance with your API credentials:

```kotlin MainActivity.kt
import com.paymaya.sdk.android.checkout.PayMayaCheckout
import com.paymaya.sdk.android.common.PayMayaEnvironment
import com.paymaya.sdk.android.common.LogLevel

class MainActivity : Activity() {
    
    private val payMayaCheckoutClient = PayMayaCheckout.newBuilder()
        .clientPublicKey("pk-NCLk7JeDbX1m22ZRMDYO9bEPowNWT5J4aNIKIbcTy2a")
        .environment(PayMayaEnvironment.SANDBOX)
        .logLevel(LogLevel.ERROR)
        .build()
        
    // ...
}
```

<ParamField path="clientPublicKey" type="string" required>
  Your PayMaya public API key. Use the Sandbox key for testing.
</ParamField>

<ParamField path="environment" type="PayMayaEnvironment" required>
  Either `SANDBOX` for testing or `PRODUCTION` for live transactions.
</ParamField>

<ParamField path="logLevel" type="LogLevel">
  Logging verbosity: `VERBOSE`, `DEBUG`, `INFO`, `WARN`, `ERROR`, or `ASSERT`. Defaults to `ERROR`.
</ParamField>

<Note>
  The example above uses a Sandbox public key. Replace it with your own key from the PayMaya Dashboard.
</Note>

## Create a checkout request

Build a `CheckoutRequest` with transaction details:

```kotlin
import com.paymaya.sdk.android.checkout.models.CheckoutRequest
import com.paymaya.sdk.android.checkout.models.Buyer
import com.paymaya.sdk.android.checkout.models.Item
import com.paymaya.sdk.android.common.models.TotalAmount
import com.paymaya.sdk.android.common.models.RedirectUrl
import java.math.BigDecimal

// Define the items being purchased
val items = listOf(
    Item(
        name = "Coffee Mug",
        quantity = 2,
        code = "MUG-001",
        description = "Ceramic coffee mug",
        totalAmount = TotalAmount(
            value = BigDecimal("500.00"),
            currency = "PHP"
        )
    )
)

// Create buyer information (optional)
val buyer = Buyer(
    firstName = "John",
    middleName = "Thomas",
    lastName = "Smith"
)

// Define redirect URLs
val redirectUrl = RedirectUrl(
    success = "https://yourapp.com/success",
    failure = "https://yourapp.com/failure",
    cancel = "https://yourapp.com/cancel"
)

// Build the checkout request
val checkoutRequest = CheckoutRequest(
    totalAmount = TotalAmount(
        value = BigDecimal("500.00"),
        currency = "PHP"
    ),
    buyer = buyer,
    items = items,
    requestReferenceNumber = "REF-${System.currentTimeMillis()}",
    redirectUrl = redirectUrl
)
```

<Warning>
  The `totalAmount` in `CheckoutRequest` should match the sum of all item amounts. Use a unique `requestReferenceNumber` for each transaction.
</Warning>

## Start the checkout flow

Launch the checkout activity with your request:

```kotlin
fun initiatePayment() {
    payMayaCheckoutClient.startCheckoutActivityForResult(this, checkoutRequest)
}
```

This opens a secure WebView where customers complete the payment process.

## Handle payment results

Override `onActivityResult` to receive the payment outcome:

```kotlin
import android.content.Intent
import android.util.Log
import android.widget.Toast
import com.paymaya.sdk.android.checkout.PayMayaCheckoutResult
import com.paymaya.sdk.android.common.exceptions.BadRequestException

override fun onActivityResult(requestCode: Int, resultCode: Int, data: Intent?) {
    payMayaCheckoutClient.onActivityResult(requestCode, resultCode, data)?.let { result ->
        when (result) {
            is PayMayaCheckoutResult.Success -> {
                val message = "Payment successful! Checkout ID: ${result.checkoutId}"
                Toast.makeText(this, message, Toast.LENGTH_LONG).show()
                Log.i(TAG, message)
            }
            
            is PayMayaCheckoutResult.Cancel -> {
                val message = "Payment canceled. Checkout ID: ${result.checkoutId}"
                Toast.makeText(this, message, Toast.LENGTH_LONG).show()
                Log.w(TAG, message)
            }
            
            is PayMayaCheckoutResult.Failure -> {
                val message = "Payment failed. Checkout ID: ${result.checkoutId}"
                Toast.makeText(this, message, Toast.LENGTH_LONG).show()
                Log.e(TAG, message, result.exception)
                
                // Handle specific error types
                if (result.exception is BadRequestException) {
                    val error = (result.exception as BadRequestException).error
                    Log.d(TAG, "Error details: $error")
                }
            }
        }
    }
}

companion object {
    private const val TAG = "MainActivity"
}
```

<Info>
  If a user closes the checkout activity (e.g., by pressing Back), the SDK automatically checks the payment status and maps it to the appropriate result type.
</Info>

## Check payment status manually

You can manually verify a payment's status using the `checkoutId`:

```kotlin
import kotlinx.coroutines.Dispatchers
import kotlinx.coroutines.withContext
import com.paymaya.sdk.android.common.CheckPaymentStatusResult

// Call from a coroutine (not the main thread)
suspend fun checkPaymentStatus(checkoutId: String) {
    val result = withContext(Dispatchers.IO) {
        payMayaCheckoutClient.checkPaymentStatus(checkoutId)
    }
    
    when (result) {
        is CheckPaymentStatusResult.Success -> {
            Log.i(TAG, "Payment status: ${result.status}")
        }
        is CheckPaymentStatusResult.Failure -> {
            Log.e(TAG, "Failed to check status", result.exception)
        }
        is CheckPaymentStatusResult.Cancel -> {
            Log.w(TAG, "Status check canceled")
        }
    }
}
```

<Warning>
  The `checkPaymentStatus` method is synchronous. Always call it from a background thread using coroutines or similar mechanisms.
</Warning>
