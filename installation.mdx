## Gradle installation

The PayMaya Android SDK v2 can be added to your Android project using Gradle. We recommend using Jitpack for the latest versions.

### Jitpack (recommended)

Jitpack hosts all current and future releases of the SDK.

<Steps>
  <Step title="Add Jitpack repository">
    Add the Jitpack repository to your root `build.gradle` file:
    
    ```groovy build.gradle
    allprojects {
        repositories {
            google()
            mavenCentral()
            maven { url 'https://jitpack.io' }
        }
    }
    ```
  </Step>
  
  <Step title="Add SDK dependency">
    Add the SDK dependency to your app module's `build.gradle` file:
    
    ```groovy app/build.gradle
    dependencies {
        implementation 'com.github.paymaya:paymaya-android-sdk-v2:<release tag>'
    }
    ```
    
    <Note>
      Check [Jitpack releases](https://jitpack.io/#paymaya/paymaya-android-sdk-v2) for the latest version tag.
    </Note>
  </Step>
  
  <Step title="Sync project">
    Sync your Gradle project to download the SDK and its dependencies.
  </Step>
</Steps>

### JCenter (deprecated)

<Warning>
  JCenter support has been deprecated. Only versions 2.0.1 and below are available in JCenter as read-only. Use Jitpack for newer versions.
</Warning>

For legacy projects still using JCenter:

```groovy build.gradle
repositories {
    jcenter()
}

dependencies {
    implementation 'com.paymaya:paymaya-sdk-android:+'
}
```

Refer to [Bintray releases](https://bintray.com/paymaya/maven-android/paymaya-sdk-android/) for available JCenter versions.

## Compatibility requirements

Ensure your project meets these minimum requirements:

<AccordionGroup>
  <Accordion title="Android OS version">
    - **Minimum SDK**: API level 23 (Android 6.0 Marshmallow)
    - **Target SDK**: API level 30 or higher
    - **Compile SDK**: API level 30 or higher
  </Accordion>
  
  <Accordion title="Kotlin version">
    The SDK is built with Kotlin 1.5.30 and requires:
    - Kotlin standard library (kotlin-stdlib-jdk8)
    - Kotlinx Serialization (1.3.0+)
    - Kotlinx Coroutines (1.3.3+)
  </Accordion>
  
  <Accordion title="AndroidX libraries">
    The SDK requires AndroidX. If you haven't migrated yet, add to `gradle.properties`:
    
    ```properties gradle.properties
    android.useAndroidX=true
    android.enableJetifier=true
    ```
  </Accordion>
  
  <Accordion title="Java compatibility">
    The SDK requires Java 8 compatibility. Add to your module's `build.gradle`:
    
    ```groovy app/build.gradle
    android {
        compileOptions {
            sourceCompatibility JavaVersion.VERSION_1_8
            targetCompatibility JavaVersion.VERSION_1_8
        }
        kotlinOptions {
            jvmTarget = "1.8"
        }
    }
    ```
  </Accordion>
</AccordionGroup>

## Required dependencies

The SDK automatically includes these dependencies:

- **OkHttp 4.3.1** - HTTP client for network requests
- **Material Components 1.2.0** - UI components
- **AndroidX AppCompat 1.2.0** - Backwards compatibility
- **AndroidX ConstraintLayout 1.1.3** - Layout management
- **Kotlinx Serialization & Coroutines** - Data serialization and async operations

## Proguard configuration

If you're using code obfuscation, the SDK includes consumer ProGuard rules automatically. No additional configuration is needed.
